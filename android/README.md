# Android native project (Capacitor 8)

Generated by `npx cap add android`. The WebView loads the same Vite React bundle as the web app (`webDir: dist`).

## Docs

- [docs/ANDROID_APK.md](../docs/ANDROID_APK.md) — build, install, troubleshoot
- [docs/MOBILE_ARCHITECTURE.md](../docs/MOBILE_ARCHITECTURE.md) — runtime architecture (`isNativeApp`, direct API URLs, mobile shell)
- In-app: `/docs/mobile/android-apk`

## Quick commands (from repo root)

```bash
npm run mobile:studio      # build + sync + open Android Studio
npm run mobile:apk:debug   # build + sync + debug APK
```

`CAPACITOR_BUILD=true` (via `npm run build:mobile`) sets Vite `base: './'` — required for asset paths in the WebView.

## Per-machine files (gitignored)

- `local.properties` — copy from `local.properties.example`; set `sdk.dir` to your SDK path
- `keystore.properties` — optional; for signed release APKs

## JDK

Gradle requires **Java 21**. `gradle.properties` points at Android Studio's bundled JBR on Windows. Adjust on macOS/Linux.

## Runtime notes

- WebView origin: `https://localhost` — add to Privy dashboard and relayer `FRONTEND_URL`
- No Vite dev proxies in APK; Zama and MedVault relayers use HTTPS URLs from `src/lib/mobile.ts`
- Package id: `xyz.medvault.app` (`capacitor.config.ts`)
